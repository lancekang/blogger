# Blogger 자동 포스팅 관리자 — 작업 진행 상황 (Current Progress)

본 문서는 프로젝트의 현재 개발 진행 상황, 최근 아키텍처 개선 내역, 정상 동작하는 주요 기능 및 남은 한계와 향후 계획을 요약한 진행 상황 리포트입니다.

---

## 1. 개발 마일스톤 및 진행 현황

본 프로젝트는 1차 기본적인 API 연동부터 시작해 현재 5차 실제 정보 수집 파이프라인 고도화 단계까지 도달해 있습니다.

| 단계 | 개발 상태 | 주요 작업 내용 |
| :--- | :---: | :--- |
| **1차: 기본 연동** | **완료** | Google OAuth 로그인 세션 구성, Blogger API 연동 (블로그 조회 및 초안/발행) |
| **2차: 에이전트 글 생성** | **완료** | 로컬 요청 큐 및 생성 결과 대기 큐 구성, Blogger 규격 인라인 스타일 가이드 적용 |
| **3차: RSS 글감** | **완료** | 카테고리별 실시간 RSS 피드 수집 API, JSDOM+Readability 기반 본문 텍스트 추출 |
| **4차: Research 파이프라인** | **완료** | Google News RSS 기반 참고자료 수집, 정규식 기반 후보명 추출 및 Placeholder 자동 차단 |
| **5차: 사실 기반 고도화** | **완료** | **RAWG 게임 DB API 연동**, **LLM 엔티티 추출**, 메타데이터 기반 AI 프롬프트 주입 (현재 상태) |

## 2. 최근 주요 개선 내역 (5차 고도화 핵심)

기존 시스템은 구글 뉴스 RSS와 단순 정규식에 의존하여 실제 게임명이나 제품명을 명확히 추출하지 못하고, AI가 존재하지 않는 가상의 정보를 생성(Hallucination)하는 문제가 있었습니다. 이를 개선하기 위해 아래의 사실 기반 수집 파이프라인을 완료했습니다.

### A. RAWG 게임 DB API 연동 (`lib/rawg.ts`)
- 게임 키워드가 감지될 경우, 구글 뉴스보다 구조화된 글로벌 게임 데이터베이스인 RAWG API를 우선 활용합니다.
- 플랫폼(Nintendo Switch, PS5, Steam 등) 및 장르(RPG, Action, Adventure 등)에 해당하는 고평가 게임 데이터를 필터링하여 정확한 메타데이터(게임명, 장르, 출시일, 평점, Metacritic 점수, 요약)를 확보합니다.

### B. 에이전트 기반 엔티티 검증
- 외부 Codex 에이전트가 공식 자료와 검색 결과를 교차 검증하여 실제 제품명과 게임명을 선별합니다.
- 웹앱 내부에서는 별도의 모델 API나 SDK를 사용하지 않습니다.

### C. 추천형 포스팅 차단 및 검증 규칙 강화
- 추천, BEST, TOP, 순위 등의 비교 분석이 필요한 키워드의 경우, 수집된 실제 후보(Candidates)가 **최소 3개 이상**이어야 포스팅 작성이 시작되도록 프론트엔드와 백엔드 양쪽에서 검증 로직을 구축했습니다.
- AI가 임의의 제품명을 만들어내지 못하도록, 수집된 Candidates 범위 안에서만 제목과 특징을 서술하도록 프롬프트 제약을 강제했습니다.

## 3. 기능별 현재 작동 상태 (Status)

| 기능 분류 | 구현된 API / 모듈 | 상태 | 비고 |
| :--- | :--- | :---: | :--- |
| **구글 로그인** | `/api/auth/[...nextauth]` / `lib/auth.ts` | **정상** | access_token 및 refresh_token 안전하게 세션 관리 |
| **블로그 조회/등록**| `/api/blogger/blogs`, `posts` / `lib/blogger.ts` | **정상** | googleapis 패키지의 커넥션 끊김 방지를 위해 fetch 기반 REST API로 전환 |
| **글감 수집 (RSS)** | `/api/sources/rss` | **정상** | IT/테크 및 게임 카테고리별 실시간 피드 조회 |
| **본문 추출** | `/api/sources/article` | **정상** | Readability를 사용한 순수 본문 5,000자 제한 추출 |
| **키워드 연구** | `/api/sources/research` | **정상** | 키워드 파싱 → RAWG 및 Google News 교차 수집 파이프라인 작동 |
| **에이전트 글 생성** | `/api/ai/request-post`, `/api/ai/pending-posts` | **정상** | 외부 에이전트에 요청을 전달하고 생성된 원고를 대기 큐로 수신 |

## 4. 현재 한계점 및 향후 개선 계획 (Roadmap)

### 1) 전자제품(IT 디바이스) 추천용 구조화 데이터 부족
- **현 상태**: 게임 키워드는 RAWG DB 덕분에 완벽한 메타데이터가 보장되나, 스마트폰/노트북/가전 등 일반 전자제품은 여전히 Google News RSS 및 LLM 엔티티 추출에만 의존하고 있어 정확한 모델명과 스펙 수집에 한계가 있습니다.
- **개선 계획**: 다나와(Danawa) 등 국내 쇼핑몰 크롤러 또는 네이버 쇼핑 Search API 등 쇼핑/리뷰 전문 API를 연동하여 전자제품 스펙 및 가격 정보의 정확도를 높일 예정입니다.

### 2) 수동 후보 편집 UI 미구현
- **현 상태**: 시스템이 수집한 Candidates가 마음에 들지 않거나 추가하고 싶은 대상이 있어도 사용자가 수동으로 가감할 수 있는 UI가 아직 존재하지 않습니다.
- **개선 계획**: AI 초안 생성을 누르기 전, 수집된 후보군 카드 옆에 [추가], [삭제] 버튼을 제공하여 최종 생성 프롬프트에 들어갈 대상을 사용자가 직접 편집할 수 있는 인터페이스를 개발할 예정입니다.

### 3) 구글 뉴스 봇 차단 대응
- **현 상태**: 특정 뉴스 사이트(조선, 동아 등 일부 메이저 언론사)의 경우 봇 방지 솔루션(Cloudflare 등)이나 페이월(Paywall)이 적용되어 있어 `@mozilla/readability`가 본문을 긁어오지 못하는 경우가 존재합니다.
- **개선 계획**: 기사 본문 추출 실패 시 meta description 기반으로 요약하거나, 프록시 IP 순환 등을 고려하고 있습니다.
